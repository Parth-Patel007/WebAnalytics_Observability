1. What is web analytics?

Web analytics is the practice of collecting and analyzing data about how people use your website or web application. Instead of guessing what users do, we measure it.

At a very basic level, web analytics answers questions like:

  • How many people visited my site today?
  • Which pages did they look at?
  • Where did they come from (search, social, ads, etc.)?
  • Did they do what I wanted them to do (sign up, buy, click)?

Key idea: analytics is about turning raw activity on your site into information you can act on.

2. Why web analytics matters

Web analytics is important because it connects your product to real user behavior. Instead of launching features and hoping they work, you can see what actually happens.

Common reasons teams care about analytics:

  • Product decisions: Which features are used most?
  • Growth & marketing: Which channels bring engaged users?
  • Conversion: Why do some users drop off before completing a goal?
  • Performance: Does a slow page hurt engagement?

Imagine you change your pricing page. With analytics, you can see whether conversions went up, down, or stayed the same after the change.

3. Key concepts & terms

Page view

A page view is counted every time a page is loaded or reloaded in the browser. If a user refreshes the page three times, that‚Äôs three page views.

User

A user is usually a unique person visiting your site. In practice, tools often use a browser cookie or similar identifier to represent a user.

Session

A session is a group of user interactions within a given time frame, often around 30 minutes of activity. One user can have multiple sessions.

Event

An event is a specific action a user takes, like clicking a button, submitting a form, or playing a video.

Conversion

A conversion is when a user completes a key goal, such as signing up, purchasing, or downloading something.

4. Types of questions analytics can answer

Good analytics setups are built around questions, not just metrics. Here are some common question types:

  • How many? How many users visited yesterday? How many sign-ups did we get?
  • Where from? Did they come from organic search, social media, or paid ads?
  • What did they do? Which pages do they view? Which buttons do they click?
  • Did it work? After changing a feature, did conversions or engagement improve?

Later in the course, we‚Äôll connect these questions to metrics in Prometheus and charts in Grafana.

5. A simple web analytics workflow

  1. Decide what you care about. (e.g., sign-ups per day)
  2. Instrument your site. Add code to send events or metrics.
  3. Collect the data. Store it in an analytics or metrics system.
  4. Visualize it. Use charts/dashboards to see trends.
  5. Act. Make changes based on what you learn, then measure again.
